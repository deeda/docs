## When to use which runtime

**`anthropic-agent-sdk`** — Use when you want Claude Code style coding and shell-native agent behavior with Anthropic models.

**`gemini-antigravity-cli`** — Use for Google Antigravity 2.0 (I/O 2026-05-19) — local agentic coding loop via the `agy` CLI + `google-antigravity` Python SDK. 11 BuiltinTools (list_directory/search_directory/find_file/view_file/create_file/edit_file/run_command/ask_question/start_subagent/generate_image/finish), lifecycle Hook classes (runtime-enumerated; the SDK self-updates), 7-primitive policy DSL (allow/deny/ask_user/allow_all/deny_all/safe_defaults/workspace_only), MCP transports resolved from the installed SDK (stdio + streamable-http today; the vendor folded SSE reads into streamable-http), file-change + interval triggers, multimodal input (Image/Document/Audio/Video). Pairs with raw Gemini API/SDK in tandem.

**`gemini-genai-sdk`** — Use when you want Gemini native streaming, Google Search, code execution, or Vertex AI-backed deployment.

**`local-llm`** — Use for managed llama.cpp or a catalog-backed local endpoint covering the V1 local universe: llama.cpp, Ollama, MLX, and LocalAI. Sprint 07 signoff is llama.cpp only.

**`local-voice`** — Use when you want local STT plus a provider-agnostic voice shell that can drive any text runtime.

**`openai-agents-sdk`** — Use when you need declarative agents, hosted tools, typed handoffs, and guardrails in the OpenAI Agents SDK.

**`openai-codex-sdk`** — Use when you want Codex CLI autonomy, workspace-aware edits, and approval-driven coding loops.

**`openai-operator`** — Use when you need computer-use style browser automation, screenshots, and hosted connectors through OpenAI tool primitives.

**`openai-realtime-api`** — Use when you need low-latency audio or multimodal Realtime sessions with server VAD and voice output.

**`openai-responses-api`** — Use when you need the broadest OpenAI knob surface, structured output, hosted tools, background runs, and webhook delivery.
